Overview Novice

A few years ago, I started a crazy project. I decided to start writing my own kernel. At the time I didn't have much knowledge of such system so I start learning a lot about system and harware principes. I already had strong skills in software development but this project really improve my knowledge and allow me to re-learn everything from a different point of view.

Features Initiated

A got scheduler, file system, a driver interface. Memory management allow mmap, heap.

Syscalls implemented:

  • sys_exit() — Quit the program and abort all related thread (implemented & tested)
  • sys_exec() — Start a new process (basic implementation)
  • sys_start() — Start a new thread (debugging)
  • sys_stop() — Put the current thread in zombie state (debugging)
  • sys_write() — Write data on a stream/file (implemented)
  • sys_read() — Read data from a stream/file (implemented)
  • sys_mmap() — Map a file on the process memory space (basic implementation)
  • sys_open() — Open a file as a used process resource (basic implementation)
  • sys_close() — Release a process resource (implemented)
  • sys_pinfo() — Load information about a process (partial implementation)
